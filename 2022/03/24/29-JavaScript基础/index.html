变量

什么叫变量？变量方便我们后续对相同的值进行修改，也方便我们对值进行使用。
变量涉及到两个概念：声明和赋值

声明变量

js中最早声明变量用var，现在都不用了。ES6中提出了新的声明变量的方式：let和const

变量赋值

一般我们写代码，声明和赋值是同时写的。

变量可以多次赋值，赋值时不要添加 let

变量命名规则

  • 变量名区分大小写，允许包含数字、字母、下划线_、美元符号$，但不能以数字开头，即第一个字符不能为数字。
  • 变量名中不能出现汉字、空格、连字符(-)、点(.)号等特殊字符。（变量名是汉字也可以运行，但是不建议）
  • 变量名不能是 JavaScript 中的关键字、保留字；
  • 变量命名长度应该尽可能的短，并抓住要点，尽量在变量名中体现存储信息的类型；
  • 尽量避免使用没有意义的命名；
关键字和保留字

关键字是js已经赋予对应的功能及含义的单词，保留字是js未来可能会赋予他们对应的功能或者含义的单词。
下面的东西了解即可，不需要记忆。
关键字：是指 JS本身已经使用了的字，不能再用它们充当变量名、方法名。
包括：break、case、catch、continue、default、delete、do、else、finally、for、function、if、in、instanceof、new、return、switch、this、throw、try、typeof、var、void、while、with、class、const 、export、import等。
保留字：实际上就是预留的“关键字”，意思是现在虽然还不是关键字，但是未来可能会成为关键字，同样不能使用它们当变量名或方法名。
包括：boolean、byte、char、debugger、double、enum、extends、fimal、float、goto、implements、int、interface、long、mative、package、private、protected、public、short、static、super、synchronized、throws、transient、volatile 等。

命名规范

驼峰命名法
当我们的单词只有一个时，全小写，如果是复合单词首字母小写，后面的每个单词首字母都大写。
帕斯卡命名法
所有的单词首字母都用大写，中间不能有符号连接

值类型

JS中有原始值类型和引用值类型。

字符串 String

这是一个很常用的值类型，可以用来展示一段文本内容。所有的字符串都是由""包裹的，既可以是双引号，也可以是单引号。

如果遇到我们对应的字符串中包含单引号或者双引号的时候，可以切换使用单双引号

如果我们的编码规范要求我们必须使用双引号或者单引号，可以使用转义符 \

布尔值 Boolean

这是我们js中一个很重要的概念，布尔类型的值只有两个，一个是true一个是false
我们使用布尔值，一般不会直接声明，我们会借助于后面要学习的条件判断，来实现一些代码的逻辑。

布尔值服务于比较运算符和一些其他的有对错结果的方法，代表的是对应的结果。

布尔值的一个很重要的概念

所有的类型都可以转换为布尔值
数字：0是false 非0是true
字符串: 空字符串 false 非空字符串true
undefined: false
null: false

Undefined

如果一个变量声明但是未赋值时会得到undefined
Undefined类型只有undefined一个值。

Null

如果一个操作应该返回一个对象，但是没有返回时，会得到一个null
undefined和null我们平时不会太常使用他们，他们存在的作用类似于NaN，目的是为了给JS中提供一个可以表示某些值的值。
undefined是null派生出来的一个值。

++ –
有些时候，我们需要我们的值+1 或者 -1，那么可以使用自增和自减操作

比较运算符

>``>=``<``<=``!=``!== ==``===
比较运算符得到的结果是布尔值，布尔值只有true 和 false
>表示大于操作，判断一个值是否比另外的值大
<表示小于操作，判断一个是否比另外的值小

== 和 ===区别

==表示相等，===表示全等，后者会比较数据类型，前者只比较值大小

未来在判断两个值是否相等时，建议用 ===更加严谨

!= 和 !== 区别

如果我们对于不等于的判断不好直接做出判断的话，我们可以取 == 和 === 的相反之

字符串的大小比较

字符串比较时根据ascii码进行比较

使用字符串中的每一位进行比较，知道比较出来大小

先比较第一位，两个都是a相同，再比较第二位，都是b，相同，再比较第三位，c a, a的ASCII码比c小，所以abc > abadsf 结果为true

逻辑运算符

&&``||``!
如果未来我们需要让我们的代码同时复合多个条件，我们可以使用一些逻辑运算符

&&

两个条件同时成立，结果才是true

比如一个数字x大于10 小于100

||

两个条件只要满足一个就成立

判断一个年费是否是闰年
闰年 能被100整除 需要能被400整除 如果不能被100整除 需要能被4整除

  • year % 100 === 0 能否被100整除
  • year % 400 === 0 能否被400整除
  • year % 100 === 0 && year % 400 === 0 两个条件需要同时满足
  • year % 4 === 0 是否能被4整除
  • year % 100 !== 0 不能被100整除
  • year % 4 === 0 && year % 100 !== 0 需要两个条件同时满足
  • 条件1 || 条件2 只要有一个满足，就满足
!

数据类型转换

我们学习到数据类型String Number Boolean Undefined Null之间可以互相转换类型，通常常用的转换就是String,Number,Boolean这三个类型的转换。

null使用typeof会得到一个object，在早期JS中null一直显示object，到ES6有意将这个结果改成null类型，后续怕影响太大，就将错就错。null到现在位置 typeof还是object

其他值转数值

转Number类型有很多方法，各种运算符的隐式转换外，还有一些显性转换。

  • Number(值)

把直接直接放在Number中，可以得到一个number类型的值，这个方法只能用来转纯数字的字符串和布尔值。

  • parseInt(值)

如果是数字或者字符串，会转换为整数，转换规则，从第一位开始，如果是数字则保留，继续第二位，直到遇到非数字的内容，停止转换
14215mdsaf -> 14215
1a111 -> 1
1.1 -> 1

  • parseFloat(值)

和parseInt相似，多取一个.遇到第二个非数字的字符就会停止。
1.121345312dsa -> 1.121345312
1.1.232 -> 1.1

  1. -> 1

这两个方法会直接把布尔值转为NaN，首位不是数字的字符串也会得到NaN
